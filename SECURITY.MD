# Security Policy

## Supported Versions

This project is currently under active development.
Only the latest version on the `main` branch is supported with security updates.

## Reporting a Vulnerability

If you discover a security vulnerability, please do **not** open a public GitHub issue.

Instead, report it responsibly by:
- Contacting the project maintainer directly (via GitHub profile or email), or
- Marking the issue clearly as **Security-related** if private reporting is not available

Please include:
- A clear description of the issue
- Steps to reproduce the vulnerability
- Potential impact (data exposure, unauthorized access, etc.)

We aim to acknowledge reports promptly and address verified issues as quickly as possible.

## Security Scope

Security concerns may include, but are not limited to:
- Unauthorized access to audio or video data
- Exposure of stored sleep recordings
- Insecure local storage of sensitive files
- Weak permissions or unsafe default configurations

Thank you for helping keep this project and its users safe.
